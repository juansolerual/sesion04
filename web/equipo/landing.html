Grupo de Trabajo

Esta actividad consiste en el desarrollo colaborativo de un sitio web para el equipo, usando repositorios GitHub para trabajar en grupo.


Sed amet aliquam

Basándose en una plantilla de sitio web estático de bootstrap o html5 (buscar ejemplos en Google), la web del equipo debe tener al menos 3 páginas html:

1. MainPage (index.html): Página con la información básica del equipo: nombre, descripción corta, enlaces a las páginas individuales, y links a recursos usados en la asignatura:

  • GitHub repositorio de este proyecto
  • GitHub official documentation
  • HMIS organization on GitHub : https://github.com/ualhmis/
  • Git book: https://git‐scm.com/book/en/v2
  • Azure portal
  • Azure official documentation

2. Páginas individuales: una página html por cada miembro del equipo, conteniendo el nombre y foto (o bien nick y avatar), enlace a la cuenta GitHub, y alguna información de gustos, aficiones, etc.El sitio web debe ser desarrollado localmente, usando su IDE favorito (SublimeText, Notepad++, Visual Studio Code, ...) trabajar en equipo, un miembro con el rol de líder, debe crear el repositorio público para el proyecto en su cuenta GitHub, y compartir la URL del repositorio con el resto del equipo. El líder debe subir al repo la página principal (index.html). Usando el modelo "fork and pull ", el resto de miembros del equipo deben "forkear" el repositorio en su propia cuenta, y proponer cambios al repositorio principal. El líder del proyecto debe aceptar los pull request. Si no recuerdas los pasos, revisa las transparencias de clase sobre “trabajo colaborativo con Git y GitHub”.
